Apa itu Mesin Turing ?

Mesin Turing adalah sebuah finte automaton yang memiliki sebuah tape tunggal dengan panjang tak terhingga yang dapat membaca dan menulis data. Mesin Turing ini awalnya diusulkan oleh Alan Turing, seorang matematikawan Inggris sebagai model matematis sederhana sebuah komputer pada tahun 1936. Mesin Turing dapat berperan sebagai mesin pengenal bahasa formal. Bahasa yang dapat dikenali Mesin Turing yaitu Unrestricted (bahasa tanpa batasan aturan produksi).

Seperti yang bisa dilihat pada gambar diatas, Mesin Turing memiliki beberapa komponen yaitu :

  1. Finite Control (Pengendali berhingga)
  2. Tape yang memiliki karakteristik :
    • Tape input panjangnya tak berhingga. Dapat dilihat pada gambar diatas dimulai dari ... sampai ... (tak berhingga)
    • Pada tape input sel yang tidak berisi simbol masukan akan berisi simbol kosong (blank = B)
    • Terdapat sebuah tape head yang ditempatkan di salah satu tape sebagai penunjuk sel yang sedang dibaca dan dapat digeser ke kiri atau kanan
    • Dapat dibaca maupun ditulis

Bagaimana Mesin Turing dilambangkan dalam notasi formal ?

Mesin Turing dalam notasi formal dilambangkan sebagai berikut :

M = (Q, σ, Γ, δ, q0, B, F)

Q: Himpunan berhingga state (a, b, c, ... atau q0, q1, q2)

Γ: Himpunan berhingga simbol yang muncul di pita

σ: Himpunan simbol-simbol, subset dari Γ, termasuk di dalamnya B

δ: Fungsi transisi yang memetakan Q × Γ → Q × Γ x {L,R}
L dan R menyatakan gerakan head left right.

qo : Status awal

F : Himpunan status akhir

Bagaimana Prinsip/Cara Kerja Mesin Turing ?

Prinsip kerja mesin turing yaitu :

1. Lihat state awal dan simbol yang ditunjuk oleh head
2. Berdasarkan fungsinya tentukan :
  1. Tentukan state berikutnya
  2. Lakukan penulisan ke tape
  3. Gerakkan head ke kanan dan ke kiri
3. Jika pasangan state dan simbol yang ditunjuk head tidak ada lagi fungsi transisinya, maka mesin turing berhenti
4. Jika mesin turing berhenti di state final berarti input diterima. Begitupun sebaliknya, jika berhenti di bukan final state maka input ditolak.

Dari grammar diatas dapat dilihat ada beberapa aturan produksi. S -> ε maksudnya nanti kita bisa mengubah Terminal menjadi sebuah string kosong. Lalu untuk i menggambarkan if dan e menggambarkan else. Berikut contohnya jika kita menurunkan dari aturan produksi grammar diatas :

Jadi dari grammar tersebut kita bisa mendapatkan string iie.


Dengan grammar diatas kita juga bisa mengecek apakah suatu string berasal dari grammar tersebut. Misalnya untuk string iieee. i pertama cocok dengan e pertama (dari sebelah kiri). Maka ini dihapus. Kemudian i kedua cocok dengan e kedua (dari sebelah kiri). Maka ini juga dihapus. Tetapi masih tersisa 1 e, sedangkan i sudah tidak ada. Sehingga string iieee ini bukan merupakan language karena tidak sesuai dengan polanya.

Contoh
Penjelasan

Hubungan dari pola grammar tersebut sesuai dengan pola if else pada C program. Dimana tidak boleh ada else yang lebih banyak dibanding if. Maksudnya if bisa muncul dengan else atau tidak, tetapi else hanya bisa ada jika ada if.


Kita telah melakukan parsing dan mendapatkan string iie yang valid dari grammar yang ada di contoh. Lalu, gambar diatas adalah implementasi string iie dalam program C. Dari pencocokan string ini dapat membantu kita dan compiler untuk mengetahui pasangan if dan else yang cocok.

Context Free Grammar dalam Markup Language

String dalam markup language adalah dokumen dengan penanda (biasa disebut tag). Tag ini menandakan semantik dari variasi string dalam document. Markup Language yang familiar diketahui yaitu HTML (Hypertext Markup Language). Di dalam html ada berbagai tag yang digunakan sesuai kebutuhan kita. Misalnya tag p untuk membuat paragraf, tag ol untuk membuat list berurutan, tag li untuk membuat list dan tag em untuk membuat teks memiliki style italic.

Variabel dalam HTML
  1. Text adalah string karakter apa pun yang dapat ditafsirkan secara harfiah, yaitu tidak memiliki tag
  2. Char adalah string apa pun yang terdiri dari satu karakter yang legal dalam teks HTML.
  3. Doc merepresentasikan documents, yang merupakan urutan-urutan dari elemen
  4. Elemen adalah salah satu string Text, atau sepasang tag yang cocok dan dokumen di antara keduanya, atau tag yang tidak cocok diikuti dengan dokumen
  5. List item adalah tag li diikuti dengan dokumen, yang merupakan item daftar tunggal.
  6. List adalah urutan dari beberapa list item
CFG dan HTML
Penjelasan

Pada no. 1 artinya char (character) bisa termasuk a, A dan sebagainya. Pada no.6 artinya list bisa kosong atau list dari list item.


Beberapa aspek HTML sebenarnya tidak butuh menggunakan context-free-grammar; karena dengan regular expression sudah cukup. Misalnya, nomor 1 dan 2 dari gambar diatas, cukup katakan bahwa Text merepresentasikan bahasa yang sama seperti halnya ekspresi reguler (a + A +). Namun, beberapa aspek HTML lainnya memang membutuhkan context free grammar. Misalnya, setiap pasangan tag yang merupakan pasangan awal dan akhir yang sesuai, contohnya seperti tag em yang harus ditutup, sama seperti () atau {} yang harus seimbang.

Kesimpulan

Pengaplikasian context free grammar bisa ditemukan pada parsing seperti penggunaan if else di program C dan juga pada markup language seperti html
