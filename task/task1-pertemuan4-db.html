Query Processing

Apa itu query ? Query bisa diartikan sebagai perintah atau bahasa yang digunakan untuk melakukan manipulasi pada database seperti menampilkan, mengupdate, menambahkan atau menghapus database. Ada beberapa step untuk pemrosesan query ini, yaitu :

01

Parsing and Translation

Pengecekan syntax dan grammar dan mengubah ke bentuk yang bisa diolah Query Processing Engine

02

Optimization

Membuat plan evaluasi yang optimal dan efisien dengan juga memperhatikan cost terendah

03

Evaluation

Memilih plan yang terbaik dari segi costnya lalu mengeksekusi query tersebut

Menghitung Biaya Query

Biaya evaluasi query bisa kita ukur dari banyaknya sumber daya sistem yang digunakan seperti pengaksesan disk, waktu CPU dan waktu komunikasi. Biasanya akses disk ini yang menjadi biaya utama, dan juga relatif mudah untuk diperkirakan. Oleh karena itu jumlah transfer blok dari disk digunakan sebagai ukuran biaya evaluasi yang sebenarnya. Biaya algoritma bergantung pada ukuran buffer di memori utama, karena memiliki lebih banyak memori mengurangi kebutuhan akses disk. Maka, ukuran memori harus menjadi parameter saat memperkirakan biaya. Untuk menghitung seberapa besar cost dari query, sebenarnya ini juga tergantung dari operasi apa yang digunakan, seberapa banyak datanya, apa saja indexnya dan beberapa aspek lain. Tetapi, berikut ini adalah beberapa cara untuk mengukur biaya query berdasarkan operasi yang digunakan.

Selection Operation

Saat melakukan operasi selection, terlebih dahulu data akan di scan dulu dengan algoritma searching lalu mengambil hasil yang ingin ditampilkan. Karena itu, cost di select operation berbeda tergantung dari bagaimana algoritma pencariannya. Beberapa contoh algoritma yang digunakan yaitu : Linear Search, Binary Search, Index Scan.

Linear Search -> cost(S1)= br
br ini menunjukkan jumlah block dari suatu relation Artinya, perkiraan costnya atau jumlah block yang discan yaitu jumlah block dari suatu tuple (baris) dari relation.

Join Operation

Beberapa metode untuk implementasi join yaitu Nested-loop join, Block nested-loop join, Indexed nested-loop join, Merge-join, Hash-join.

Misalkan untuk nested-loop join,yang menggabungkan tabel dengan membandingkan nilai di tabel yang satunya (outer table) dengan tabel kedua (inner table) lalu mencari baris yang memenuhi persyaratan joinnya, biayanya untuk worst case yaitu : nr âˆ— bs + br.
Artinya dengan menggunakan rumus diatas dapat dihitung block transfer yang dibutuhkan dengan keterangan : nr = jumlah tuple di tabel outer, br = jumlah block di tabel outer dan bs = jumlah block di tabel inner

Selain kedua operation diatas, ada beberapa operation lainnya yang biasa digunakan untuk menghitung cost dari query seperti duplicate elimination, projection dan set operation (union, intersection).



Bagaimana Query Mengevaluasi Ekspresi ?

Untuk mengevaluasi/memproses ekspresi baik sebuah ekspresi ataupun multiple ekspresi, query menggunakan kedua metode berikut :

Materialisasi

Dengan metode materialisasi ini, query-query akan dipisahkan/dibagi dulu menjadi query individual. Kemudian, akan dievaluasi querynya dan disimpan di tabel sementara pada memori. Data dari tabel sementara ini, akan digunakan untuk mengevaluasi query selanjutnya. Tetapi, walaupun terlihat simple, metode ini membutuhkan biaya besar untuk evaluasinya karena pastinya butuh waktu untuk evaluasi dan menuliskan data ke tabel sementara.

Pipeline

Di metode ini, setiap operasi relational dapat dievaluasi secara bersamaan pada pipeline. Jadi, misalkan sebuah query akan dievaluasi terlebih dahulu lalu hasilnya akan digunakan untuk query berikutnya dan begitu seterusnya dimana hasil evaluasi query sebelumnya, dilanjutkan ke query berikutnya. Karena itu, di metode ini, tidak lagi disimpan di tabel sementara yang berdampak ke berkurangnya cost. Mengapa ? Karena tidak ada lagi cost untuk menulis dan mengakses data ke tabel sementara.
